왜 중요한가

대부분의 사용자가 내리는 가장 어려운 결정

웨딩 플래닝은 고위험·저구조·파편화된 정보로 가득합니다.

01 — 탐색 문제

기존 플랫폼은 SEO 기반 — 추천 기반이 아닙니다

웨딩 업체 플랫폼은 키워드 매칭과 유료 노출 기준으로 결과를 제공합니다. "서울 야외 소규모 베뉴"를 검색하면 관련 없는 결과가 가득합니다. 신호는 있지만 시스템이 읽지 못합니다.

02 — AI 공백

일반 검색은 키워드를 반환합니다. 사용자는 구조화된 의도 이해가 필요합니다.

사용자가 실제로 말하는 것: "야외, 200명, 프리미엄 느낌, 6천만 원 이하." 키워드 검색이 하는 것: 실패. LLM 필터 추출이 있는 RAG 파이프라인이 하는 것: 검색 실행 전에 위치·인원·스타일·예산을 구조화된 파라미터로 추출.

⚡

핵심 기회: 기업 검색에서 사용하는 것과 동일한 인텔리전스 레이어를 파편화된 소비자 마켓플레이스에 적용 — 모호한 쿼리를 확신 있는 구조화된 결정으로.

데모 시나리오

고객이 3분 안에 보는 것

자연어 쿼리부터 구조화된 비교까지 — 라이브로 완전히 보여줄 수 있습니다.

1
사용자 입력

"서울 야외 베뉴, 300명, 예산 8천만 원 이하"

사용자가 자연스러운 대화형 쿼리를 입력합니다. 폼 필드도, 드롭다운도 없습니다. 구조화는 시스템이 처리합니다.

2
LLM 필터 추출

검색 실행 전에 조건이 파싱됩니다

LLM이 추출합니다: 위치=서울, 유형=야외, 수용인원≥300, 예산≤8천만 원. 구조화된 필터 준비 완료 — 임베딩 생성 전에 검색 범위 확정.

3
벡터 검색

쿼리 임베딩이 업체 데이터베이스와 매칭됩니다

pgvector 유사도 검색으로 쿼리 의도와 의미적으로 일치하는 업체를 발견 — 키워드 중복이 아닌 의미 기반. 추출된 조건으로 동시에 필터링.

4
재순위

시맨틱 + 조건 매칭 결합 점수로 결과 순위 결정

후보 결과가 결합 점수로 재순위됩니다: 시맨틱 유사도 + 조건 충족도. 둘 다 맞는 업체가 상위에. 부분 매칭은 숨기지 않고 하위 순위로.

5
구조화된 출력

사용자가 구조화된 비교로 순위 결과를 확인합니다

결과 표시: 이름, 수용 인원, 가격대, 스타일 태그, 매칭 점수. 사용자가 10개 웹사이트를 돌아다니지 않고도 비교하고 결정할 수 있습니다. 쿼리 하나가 수시간의 수동 조사를 대체했습니다.

비즈니스 가치

세 명의 구매자. 하나의 엔진.

동일한 RAG 인프라가 각 이해관계자에게 유료 가치를 창출합니다.

💍

웨딩 플랫폼

업체 매칭 전환율 향상

  • AI 탐색 레이어가 키워드 검색을 대체
  • 관련 업체를 빠르게 발견 → 이탈률 감소
  • 구글 검색 대비 플랫폼 점착성 증가
  • 구조화된 결과 → 더 많은 문의 제출
🏪

마켓플레이스 운영사

임베드 가능한 추천 엔진

  • RAG 파이프라인을 고선택 버티컬 전반에 적용 가능
  • NL 필터 추출이 카테고리 간 재사용 가능
  • pgvector + Supabase 스택의 비용 효율성
  • 아직 키워드 검색을 쓰는 플랫폼 대비 차별화
🧠

B2B SaaS 벤더

의사결정 지원을 위한 검증된 RAG 아키텍처

  • 쿼리부터 순위 결과까지 end-to-end 파이프라인
  • 구조화된 검색을 위한 LLM + 벡터 DB 패턴
  • 고위험 맥락에서 AI 지원 결정 실증
  • 부동산·채용·여행 등으로 복제 가능

출시가 아닌 판매를 위해 만들었습니다
