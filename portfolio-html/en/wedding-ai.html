Why This Matters

The hardest decision most users ever make

Wedding planning is high-stakes, low-structure, and full of fragmented information.

01 — The Discovery Problem

Existing platforms are SEO-driven, not recommendation-driven

Wedding vendor platforms surface results based on keyword matches and paid placement — not user intent. A user searching for "intimate outdoor venue in Seoul" gets pages of irrelevant results. The signal is there; the system can't read it.

02 — The AI Gap

Generic search returns keywords. Users need structured intent understanding.

What users actually say: "outdoor venue, 200 guests, feels premium, under ₩60M." What keyword search does with that: misses. What a RAG pipeline with LLM filter extraction does: extracts location, capacity, style, and budget as structured search parameters before the search even runs.

⚡

The real opportunity: apply the same intelligence layer used in enterprise search to a fragmented consumer marketplace — turning ambiguous queries into confident, structured decisions.

Demo Scenario

What a client sees in 3 minutes

From a natural language query to a structured comparison — fully demoable live.

1
User Input

"Outdoor venue in Seoul, 300 guests, budget under ₩80M"

User types a natural, conversational query. No form fields, no dropdowns. The system handles the structuring.

2
LLM Filter Extraction

Constraints parsed before search runs

LLM extracts: location = Seoul, type = outdoor, capacity ≥ 300, budget ≤ ₩80M. Structured filters ready — search knows what to look for before embedding even generates.

3
Vector Search

Query embedding runs against vendor database

pgvector similarity search finds vendors whose descriptions match the semantic intent of the query — not just keyword overlap. Filtered by the extracted constraints simultaneously.

4
Re-ranking

Results scored by combined semantic + constraint match

Candidates re-ranked by a combined score: semantic similarity + constraint satisfaction. Vendors that match both rise to the top. Partial matches ranked lower, not hidden.

5
Structured Output

User sees ranked vendors with structured comparison

Results show: name, capacity, price range, style tags, match score. User can compare and decide without visiting 10 different websites. One query just replaced hours of manual research.
