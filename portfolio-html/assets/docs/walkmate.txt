1. Sales Engineer로 보이게 만드는 핵심 추출 포인트
1) 고객 문제를 기술 문제로 번역한 경험

WalkMate는 단순히 “시각장애인 앱”이 아니라,
현장 문제를 발견하고, 기술 구조로 재정의한 사례로 해석할 수 있습니다.

추출 포인트:

기존 보조도구의 한계 파악: 발밑 중심 안내, 전방 위험물 감지 부족
사용자 문제를 기능으로 분해:
실시간 장애물 탐지
경로 이탈 복귀 안내
무터치 음성 UX
관리자 모니터링 시스템
기술 선택이 문제 해결과 직접 연결됨:
온디바이스 AI = 지연 최소화
Tmap 기반 내비게이션 = 실제 이동 경로 제공
PostGIS 기반 위험 데이터 축적 = 운영/행정 활용 가능성 확보

이건 sales engineer 관점에서 매우 중요합니다.
왜냐하면 세일즈 엔지니어는 제품 기능을 나열하는 사람이 아니라,
고객의 pain point를 기술 솔루션 구조로 바꾸는 사람이기 때문입니다.

발표 자료에서도 “기존 보조 도구는 발밑만 보고, WalkMate는 전방을 본다”는 식으로 문제와 차별점을 분명히 정의하고 있습니다

2) 기술을 비기술 이해관계자도 이해할 수 있게 설명한 경험

세일즈 엔지니어에게 매우 중요한 역량입니다.

추출 포인트:

기술 설명을 사용자/기관 관점 가치로 번역
“온디바이스 AI” → 네트워크가 끊겨도 실시간 탐지 지속
“PostGIS 기반 DB” → 위험 구역 데이터 축적 및 행정기관 활용 가능
“Priority Queue TTS” → 오디오 충돌 방지, 사용자 피로도 감소
기능이 아니라 결과 중심으로 설명
실시간성
안전성
운영 효율
데이터 기반 의사결정

발표 자료 자체가 이미 기술 발표이면서 동시에 “설득용 자료” 구조를 가지고 있습니다.
특히 솔루션 소개, 관리자 대시보드, ROI, 수익모델 파트는 전형적으로 sales/demo 자료로 전환하기 좋습니다

3) 단순 개발자가 아니라 ‘솔루션 제안형’ 역할을 수행한 경험

윤지현님의 역할은 발표 자료상 팀장 · Frontend로 명시되어 있고,
Capacitor+React 하이브리드 앱 구축, Tmap API, Voice UI, 센서 퓨전, TTS 큐 설계 등이 포함되어 있습니다

세일즈 엔지니어 관점으로 바꾸면 이렇게 됩니다.

추출 포인트:

고객 접점이 가장 강한 사용자 앱 UX 설계
실제 사용 환경(걷는 중, 손 사용 어려움, 소음, 방향 감각 저하)을 반영한 인터랙션 설계
기능 구현을 넘어 “실사용성” 중심으로 조정
여러 기술 요소를 묶어 하나의 사용자 가치로 통합
STT
TTS
GPS/방향 안정화
내비게이션
위험 감지 알림

이건 “고객 데모를 설계할 수 있는 사람”, “기술 PoC를 고객 사용 시나리오 기준으로 구성할 수 있는 사람”으로 연결됩니다.

4) PoC / 데모 / 프리세일즈형 역량

세일즈 엔지니어는 종종 완성품보다 PoC, 데모, 기술 검증을 다룹니다.
WalkMate는 이 점에서 상당히 강합니다.

추출 포인트:

실제 동작 가능한 모바일 앱 + 관리자 대시보드 + 백엔드 파이프라인까지 연결
end-to-end 흐름 설명 가능
탐지
이미지 업로드
메타데이터 저장
지도 시각화
상태 관리
“기술이 돌아간다”를 보여주는 데모 구조 보유
성능/안정성/운영성에 대한 설명 가능
1초 미만 루프
응답 속도 개선
로그 엔드포인트
예외처리
Zero-refresh dashboard

이건 세일즈 엔지니어 채용에서 매우 좋습니다.
왜냐하면 많은 지원자가 “기능 구현”까지만 말하고,
실제 고객 앞에서 보여줄 데모 스토리까지는 못 만들기 때문입니다.

발표 자료의 관리자 대시보드, 백엔드 파이프라인, 로깅 시스템 부분이 이 근거가 됩니다

5) 비즈니스 관점까지 이해한 경험

이 부분이 특히 강점입니다.
보통 주니어 개발 포트폴리오는 기술까지만 있고,
WalkMate는 시장 규모, 수익 구조, 파트너십 로드맵, ROI까지 들어가 있습니다

세일즈 엔지니어 관점 추출 포인트:

TAM / SAM / SOM 구조 이해
고객군 세분화
PM 업체
지자체
ESG 후원 대기업
고객별 가치 제안 차별화
PM 업체: 견인비 절감
지자체: 행정비 절감
대기업: ESG 임팩트
기술을 매출 구조로 연결하는 사고
ROI 관점으로 솔루션 판매 논리 구성

이건 매우 중요합니다.
세일즈 엔지니어는 기술만 아는 사람이 아니라
“왜 고객이 돈을 내야 하는가”를 기술과 연결할 수 있어야 하기 때문입니다.

6) 이해관계자별 커뮤니케이션 역량

WalkMate는 사용자만 상대하는 제품이 아닙니다.

추출 포인트:

최종 사용자: 시각장애인 / 보행 약자
운영자: 관리자 대시보드 사용자
기관/기업: 지자체, PM 업체, ESG 기업
팀 내부: AI / Backend / Frontend 협업
멘토 피드백 반영: API 구조 리팩토링

즉, 하나의 기술을 여러 이해관계자 언어로 바꾸는 경험이 있습니다.
이건 sales engineer 업무와 매우 가깝습니다.

발표 자료에서도 멘토링 피드백 반영, RESTful 리팩토링, 관리자용 기능 설계가 명시되어 있습니다

7) 기술 트러블슈팅을 고객 가치 관점으로 연결한 경험

세일즈 엔지니어는 단순히 장애를 고치는 사람이 아니라,
기술 문제를 고객 신뢰 문제로 해석할 수 있어야 합니다.

WalkMate에서 추출할 수 있는 포인트:

점자블록 탐지 실패 → 방향 전환(PIVOT)로 문제 정의 재설정
TTS 중복 → 사용자 피로도 문제 해결
저속 보행 시 화살표 떨림 → 센서 퓨전으로 UX 안정화
경로 이탈/체크포인트 문제 → Reverse Search로 안내 신뢰성 개선
API 구조 문제 → 프론트 연동성과 통신 안정성 개선
S3 권한 오류 → 실제 운영환경에서 자주 생기는 배포 이슈 해결

이런 내용은 포트폴리오에서 반드시 강조해야 합니다.
세일즈 엔지니어는 데모 중 문제가 생겼을 때도 기술/비즈니스 영향도를 같이 설명해야 하므로,
이런 식의 사고가 강하게 어필됩니다

2. 안티그래비티 에이전트에 넘길 “추출 결과” 형태

아래는 그대로 넘기기 좋은 정리본입니다.

WalkMate에서 Sales Engineer 포트폴리오용으로 강조할 역량
A. Problem Discovery & Solution Mapping
시각장애인 보행 환경의 실제 문제를 분석하고, 이를 기술 구조로 재정의함
기존 보조도구의 한계(발밑 중심, 전방 장애물 인지 부족)를 파악해 차별화된 솔루션 제안
사용자 문제를 기능 단위가 아니라 end-to-end 경험 단위로 설계
실시간 장애물 탐지
음성 기반 내비게이션
관리자용 모니터링 시스템
도시 안전 데이터 축적
B. Customer-Facing Technical Communication
복잡한 기술 요소를 비기술 이해관계자도 이해할 수 있는 가치 중심 언어로 설명
온디바이스 AI, PostGIS, TTS Queue, Sensor Fusion 같은 기술을
“지연 최소화”, “운영 효율”, “사용자 안전”, “데이터 활용성”으로 번역
발표 자료를 기술 설명이 아닌 설득형 솔루션 제안서 형태로 구성
C. Pre-Sales / Demo / PoC Capability
모바일 앱, 백엔드, 관리자 대시보드를 연결한 end-to-end 데모 환경 구축
고객이 실제 사용 시나리오를 상상할 수 있는 흐름 중심으로 솔루션 구성
기술 검증 수준을 넘어서 실제 도입 가능성을 설명할 수 있는 PoC 형태 구현
D. UX-Centric Technical Design
팀장 및 프론트엔드 담당으로서 사용자 접점이 강한 UX를 직접 설계
무터치 음성 인터페이스, 방향 안정화, 우선순위 음성 큐 등
실제 사용 맥락을 반영한 기능 설계
기능 구현보다 사용성, 이해도, 접근성을 우선한 설계 경험 보유
E. Business & Value Proposition Thinking
기술 구현에 그치지 않고 TAM/SAM/SOM, 수익 구조, 파트너십 모델, ROI까지 설계
고객군별 가치 제안을 구분해 설명 가능
PM 업체: 견인비 및 운영비 절감
지자체: 위험구역 관리 및 행정 효율화
대기업: ESG 가치 및 사회적 임팩트
기술을 매출 구조와 연결하는 사고를 보여줌
F. Cross-Functional Collaboration
Frontend, Backend, AI 역할이 분리된 팀 구조에서 팀장으로 협업 조율
멘토 피드백을 반영해 API 구조를 개선하고 연동 방식을 재정렬
기술 구현뿐 아니라 커뮤니케이션과 조율을 통해 솔루션 완성도를 끌어올림
G. Technical Troubleshooting with Customer Impact Awareness
기술 문제를 단순 버그가 아니라 사용자 신뢰와 운영 안정성 문제로 보고 해결
탐지 실패 시 방향 전환(PIVOT), 센서 불안정 시 Fusion 적용, TTS 중복 시 Queue 설계
API 설계, S3 권한, 예외 처리 등 실제 도입/운영 환경에서 발생하는 문제 해결 경험 보유
3. 포트폴리오 문장으로 바꾸면 좋은 표현

안티그래비티가 포트폴리오를 만들 때 아래 표현 축으로 쓰면 좋습니다.

한 줄 포지셔닝
기술을 고객 가치로 번역하는 문제 해결형 Sales Engineer 지향 인재
사용자 문제를 end-to-end 솔루션으로 구조화할 수 있는 기술 기반 커뮤니케이터
PoC, 데모, UX, 운영 구조까지 연결해 설명할 수 있는 솔루션형 엔지니어
WalkMate 프로젝트 요약 문장
WalkMate는 시각장애인의 보행 안전 문제를 해결하기 위해 온디바이스 AI, 음성 UX, 내비게이션, 관리자 대시보드를 통합한 솔루션으로, 단순 기능 개발이 아니라 고객 문제 정의부터 기술 구조화, 비즈니스 활용성까지 설계한 프로젝트였습니다.
프로젝트 팀장 및 프론트엔드 담당으로서 사용자 접점 설계와 데모 흐름 구축을 주도했고, 기술을 실제 사용성과 도입 가능성의 언어로 설명하는 역량을 키웠습니다.
특히 사용자·운영자·기관 각각의 관점에서 다른 가치 제안을 구성하며, 세일즈 엔지니어 역할에 필요한 기술 커뮤니케이션과 솔루션 제안 역량을 실전형으로 경험했습니다.
4. 포트폴리오에서 반드시 들어가야 할 섹션명

안티그래비티 에이전트에게는 이런 구조를 요구하면 좋습니다.
